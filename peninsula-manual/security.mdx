## Encryption Standards

All data is encrypted in transit and at rest using industry-standard protocols.

| Layer | Standard | Details |
| --- | --- | --- |
| **In Transit** | TLS 1.3 | All API calls, data transfers, and user sessions |
| **At Rest** | AES-256 | All stored data, indexes, and backups |
| **Key Management** | HSM-backed | Hardware security modules for key storage |
| **Database** | Transparent encryption | Column-level encryption for sensitive fields |

**Certificate management:**
- Automated certificate rotation
- Certificate transparency logging
- HSTS enforcement on all endpoints

## Zero Data Retention Policy

Peninsula operates with a zero data retention policy for AI model interactions.

<Note>
Your data is never used to train AI models. Prompts and responses are not retained by model providers.
</Note>

**What this means:**
- Queries sent to AI models are not stored by the model provider
- No training on your firm's data or conversations
- Session data is ephemeral and cleared after processing
- You maintain full ownership and control of your data

**Data lifecycle:**

| Stage | Retention | Control |
| --- | --- | --- |
| Source data | Per your policy | You control retention in source systems |
| Indexed data | Per your policy | Configurable retention and deletion |
| AI interactions | Zero retention | Not stored by model providers |
| Audit logs | Configurable | Typically 1-7 years per compliance needs |

## Access Controls & RBAC

Peninsula enforces role-based access control (RBAC) at every layer.

### Permission Model

<CardGroup cols={2}>
  <Card title="Role-Based Access" icon="user-shield">
    Permissions assigned by role, not individual user
  </Card>
  <Card title="Matter-Level Restrictions" icon="folder-closed">
    Data access scoped to specific matters and clients
  </Card>
  <Card title="Least Privilege" icon="lock">
    Users receive minimum permissions needed for their role
  </Card>
  <Card title="Inheritance Controls" icon="sitemap">
    Permissions flow from source systems with override capability
  </Card>
</CardGroup>

### Standard Roles

| Role | Access Level | Typical Users |
| --- | --- | --- |
| **Viewer** | Read-only access to assigned matters | Support staff, reviewers |
| **Contributor** | Read/write within assigned matters | Associates, paralegals |
| **Manager** | Full access plus team management | Partners, practice leads |
| **Admin** | System configuration and user management | IT administrators |

### Access Enforcement

- Permissions checked before every retrieval operation
- ACLs propagated from source systems
- Real-time permission updates (no caching delays)
- Audit logging of all access attempts

## Data Handling Policies

### Data Classification

Peninsula supports configurable data classification tiers:

| Tier | Description | Controls |
| --- | --- | --- |
| **Public** | Non-sensitive, shareable | Standard access controls |
| **Internal** | Business-sensitive | Role-based access |
| **Confidential** | Client matter data | Matter-level restrictions |
| **Restricted** | Highly sensitive | Named-user access only |

### Data Processing

- Processing occurs only in designated regions
- No data leaves your configured geography
- Sub-processors disclosed and contractually bound
- Data processing agreements (DPAs) available

### Data Deletion

- Self-service deletion for authorized users
- Automated retention policy enforcement
- Cryptographic deletion for indexed data
- Deletion certificates available on request
